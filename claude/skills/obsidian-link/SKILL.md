# 옵시디언 노트 링크 연결

사용자가 `/연결` 을 입력하면 아래 절차를 따른다.

## 실행 절차

### 1단계: 모드 선택 (AskUserQuestion 사용)

반드시 AskUserQuestion 도구로 아래 질문을 한다:

**질문**: "어떤 방식으로 링크를 연결할까요?"

**옵션** (이 순서로 제시):
1. **방금 추가한 노트** — 가장 최근에 수정된 노트를 볼트 전체와 대조해서 연결 (기본)
2. **특정 노트** — 파일명이나 경로를 알려주면 해당 노트만 연결
3. **전체 스캔** — 볼트 전체 노트를 분석해서 Related 섹션 일괄 업데이트

---

### 2단계: 모드별 실행

#### 모드 1 — 방금 추가한 노트 (기본)

```
1. 볼트 경로: /Volumes/Data_Storage/Documents/Agent Vault/Second Brain
2. find 명령으로 최근 수정된 노트 상위 5개 확인
3. 사용자에게 "이 노트가 맞나요?" 확인
4. 해당 노트 Read → 내용 분석
5. 볼트 전체에서 관련 노트 검색 (태그, 주제, 키워드 기준)
6. Related 섹션에 연관 노트 [[링크]] 추가
7. 연결된 상대 노트의 Related에도 역방향 링크 추가
```

#### 모드 2 — 특정 노트

```
1. 사용자에게 파일명 확인 (이미 알고 있으면 바로 진행)
2. 해당 노트 Read → 내용 분석
3. 볼트 전체에서 관련 노트 검색
4. Related 섹션 업데이트
5. 역방향 링크 추가
```

#### 모드 3 — 전체 스캔

```
1. 볼트 전체 노트 목록 수집 (Templates, System 제외)
2. Related 섹션이 비어있거나 [[]] 만 있는 노트 우선
3. Home.md Orphans 섹션 기준으로 우선순위 결정
4. 노트별로 내용 분석 → 관련 노트 연결
5. 처리 결과 요약 보고
```

---

### 3단계: 링크 연결 기준

연관 노트를 판단하는 기준 (이 순서로):

1. **같은 태그** — `topic/ai` 등 주제 태그가 겹치는 노트
2. **같은 Projects↔Areas** — 번호가 같은 폴더 (예: 11_AI ↔ 21_AI)
3. **내용 키워드 겹침** — 핵심 개념, 인물, 도구 이름이 겹치는 노트
4. **시간적 연관** — 같은 시기 작성된 같은 주제 노트

**링크 추가 규칙**:
- 확실히 연관된 것만 추가 (애매하면 추가하지 않음)
- 기존 Related 섹션의 내용은 건드리지 않음 (추가만)
- `[[]]` 빈 링크는 제거하고 실제 링크로 교체
- 역방향 링크도 함께 추가

- 역방향 링크 추가 현황
