# Scripts Rules
> 루트 CLAUDE.md의 모든 규칙을 상속하며, 충돌 시 이 파일이 우선한다.

## Scope
스크립트, CLI, 배치, 데이터 처리, 자동화 작업에 적용한다.

## Before coding
- 입력, 출력, 실행 환경, 재실행 가능성을 먼저 적는다.
- 일회성 스크립트인지 반복 실행 자동화인지 먼저 구분한다.
- 일회성이라도 "다시 돌렸을 때 깨지는가?"를 확인한다.

## Change rules
- 반복 실행 스크립트는 가능하면 idempotent 하게 만든다.
- 경로, 인코딩, 시간대, 환경 변수 의존성을 먼저 확인한다.
- 조용히 실패하지 않게 한다 — 실패 시 명확한 에러 메시지를 출력한다.
- 요청받지 않은 설정 시스템이나 프레임워크를 추가하지 않는다.
- 샘플 입력 하나로라도 바로 검증 가능한 형태를 선호한다.

## Verification
- 샘플 입력/출력으로 먼저 확인한다.
- 확인 항목: 파일 생성, 덮어쓰기 동작, 예외 처리, 로그 메시지
- 가능하면 dry-run 옵션을 제공하여 파괴적 동작 전에 미리보기를 지원한다.
