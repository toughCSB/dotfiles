# Frontend Rules
> 루트 CLAUDE.md의 모든 규칙을 상속하며, 충돌 시 이 파일이 우선한다.

## Scope
화면, 컴포넌트, 스타일, 폼, 상태, 사용자 상호작용 변경에 적용한다.

## Before changing UI
- 사용자가 실제로 겪는 문제를 한 줄로 적는다.
- 이 변경이 텍스트 수정인지, 동작 수정인지, 구조 수정인지 먼저 구분한다.
- 기존 컴포넌트, 공용 스타일, 디자인 토큰이 있으면 우선 사용한다.
- 새 컴포넌트를 만들기 전에 기존 것으로 해결 가능한지 확인한다.

## Change rules
- 작은 UI 문제 때문에 구조를 과하게 뒤엎지 않는다.
- 요청받지 않은 시각적 개선을 추가하지 않는다.
- 한 번만 쓰는 UI를 위해 공용 추상화를 만들지 않는다.
- 아래 상태 중 빠뜨린 것이 없는지 확인한다:
  - 정상 표시 / 로딩 / 에러 / 빈 상태
- 접근성(a11y)이나 키보드 동작에 영향이 있으면 같이 확인한다.

## Verification
- 변경 전/후 사용자 관점에서 무엇이 달라졌는지 한 줄로 적는다.
- 가능하면 실제 클릭/입력 흐름으로 확인한다.
- 확인 항목: 버튼 상태, 폼 검증, 에러 메시지, 반응형 깨짐 여부
