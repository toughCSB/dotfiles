staged 변경 사항을 분석하고 한국어 커밋 메시지를 만들어 커밋합니다.

## 실행 절차

1. `git diff --staged` 로 staged 변경 확인
2. staged가 없으면 관련 파일을 `git add` 제안
3. 변경 내용 분석 후 아래 형식으로 메시지 제안:
   ```
   유형: 요약 (50자 이내)
   ```
   유형 목록: `기능` `수정` `리팩터` `문서` `테스트` `설정`
4. 예시: `수정: 이메일 유효성 검증 로직 추가`
5. 승인 후 커밋 실행

## 규칙
- 한 커밋 = 하나의 관심사
- 관련 없는 변경은 분리 제안
- `.env`, 인증 정보가 포함되면 반드시 경고
