## 디버깅 절차

### 1단계: 재현
- 에러가 발생하는 정확한 조건 파악
- 최소 재현 케이스 작성 시도

### 2단계: 원인 격리
- 에러 메시지/스택 트레이스 분석
- 관련 파일, 함수, 데이터 흐름 추적
- 최근 변경 사항 확인 (`git log --oneline -10`)

### 3단계: 가설 수립
- 가능한 원인을 우선순위 순으로 나열
- 가장 단순한 가설부터 검증

### 4단계: 수정
- 최소한의 변경으로 수정
- 수정 후 같은 경로로 재확인

### 5단계: 리그레션 방지
- 재발 방지를 위한 테스트 추가 제안

## 규칙
- 같은 접근 2회 실패 시 다른 방법으로 전환
- 3회 실패 시 멈추고 상황 보고
- 추측으로 "고쳤다"고 하지 않음
